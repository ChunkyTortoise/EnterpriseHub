# Portfolio Project Discovery Framework

## Strategic Discovery Questions

### Market Positioning
1. **Service Catalog Context**
   - What specific expertise/certification does this project showcase?
   - What's your highest-value service offering this demonstrates?
   - Who is your ideal high-ticket client for this expertise?

2. **Competitive Advantage**
   - What makes your approach unique in the market?
   - What complex problems do you solve that others don't?
   - What's your typical project value range for this type of work?

3. **Business Impact**
   - What measurable outcomes do your clients achieve?
   - What ROI do clients typically see from your services?
   - What case studies or success stories can this project reflect?

### Technical Demonstration
4. **Core Capabilities**
   - What technical skills does this project need to showcase?
   - What industry-specific knowledge should be evident?
   - What level of complexity demonstrates your expertise level?

5. **Integration Complexity**
   - What systems/platforms should this integrate with?
   - What APIs, databases, or external services are relevant?
   - What scalability considerations demonstrate enterprise readiness?

6. **User Experience**
   - Who will be evaluating this project (technical vs business stakeholders)?
   - What workflow should the demo showcase?
   - What "wow factor" features will differentiate this portfolio piece?

### Project Scope
7. **MVP Definition**
   - What's the minimum viable demonstration of your expertise?
   - What features are essential vs nice-to-have for portfolio impact?
   - What can be simulated vs what needs to be fully functional?

8. **Development Constraints**
   - What's your target timeline for this portfolio piece?
   - What existing code/frameworks can be leveraged?
   - What external dependencies should be minimized?

9. **Success Metrics**
   - How will you measure this project's impact on your business?
   - What client conversations should this project enable?
   - What pricing tier should this project justify?

## Follow-Up Deep Dive Areas

Based on initial responses, explore these areas:

### If SaaS/Product Focus:
- Revenue model demonstration
- User onboarding experience
- Data visualization/reporting capabilities
- API design and documentation

### If Consulting/Services Focus:
- Problem-solving methodology
- Framework/process demonstration
- Industry-specific customization
- Scalability and efficiency gains

### If Technical/Integration Focus:
- Architecture patterns and best practices
- Performance optimization examples
- Security implementation
- Monitoring and observability
