**Interesting Challenge**:
The team faced a tricky race condition where concurrent user swipes on properties could create duplicate cache entries. Traditional debugging would have taken hours.

Instead, Maya prompted: "Fix race condition in property swipe caching"

The `condition-based-waiting` skill:
1. Detected the race condition automatically
2. Implemented a distributed lock using Redis
3. Added tests proving the fix worked under 100 concurrent requests
4. Documented the solution for future reference

Total time: 8 minutes.

**Code Sample** (Before/After):
```python
# BEFORE: Naive caching (race condition)
def cache_swipe(user_id, property_id, action):
    key = f"swipe:{user_id}:{property_id}"
    redis.set(key, action)  # ⚠️ Race condition!

# AFTER: With distributed lock (skill-generated)
async def cache_swipe(user_id, property_id, action):
    key = f"swipe:{user_id}:{property_id}"
    lock_key = f"lock:{key}"

    async with redis.lock(lock_key, timeout=5):
        # Check if already exists (double-check pattern)
        existing = await redis.get(key)
        if existing:
            return existing

        await redis.set(key, action, ex=3600)
        return action
```
